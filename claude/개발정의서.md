- 본 개발정의서를 단일 기준으로 유지
- API 계약 및 상태 머신 정의
- DB 스키마 최종 확정

작업:
1. /docs/api-contracts.md 작성
2. /docs/agent-protocol.md 작성
3. tasks 상태 머신 정의
4. breaking change 금지
2️⃣ Supabase DB Agent
당신은 Supabase DB/RLS 담당입니다.

목표:
- 멀티테넌트 구조 구현
- org 기반 RLS 격리

작업:
- migrations 생성
- organizations, projects, tasks, documents 테이블 구현
- RLS 정책 적용
- storage bucket 정책 정의

규칙:
